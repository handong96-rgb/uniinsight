# 하네스 팀 오케스트레이터 (Team Orchestrator)

본 문서는 Power BI & Gemini AI Insight 팀의 협업 프로세스를 관리하는 메타 지침입니다.

## 🔄 표준 워크플로우

### 1단계: 데이터 아키텍처 수립 (Data Architect)
- Power BI 세매틱 모델 스캔 및 `schema_map.json` 업데이트.
- 분석에 필요한 DAX 쿼리 셋업.

### 2단계: 분석 프롬프트 및 로직 설계 (Intelligence Specialist)
- **Gemini 3 Flash Preview**를 위한 시스템 지침 고도화.
- '19계명' 기반의 리포트 구조 확립.

### 3단계: 사용자 인터페이스 구현 (Frontend Artisan)
- 웹페이지 UI 개발 및 AI 분석 API 연동.
- 프리미엄 디자인 시스템(Glassmorphism) 적용.

### 4단계: 종합 품질 검증 (QA Specialist)
- 데이터 수치 대조 및 19계명 준수 여부 전수 조사.
- UI/UX 버그 픽스 및 최종 승인.

## 📡 커뮤니케이션 규칙
- 에이전트 간의 모든 소통은 마크다운 기반의 아티팩트와 로그를 통해 기록됩니다.
- 이슈 발생 시 즉시 `implementation_plan.md`를 업데이트하여 팀 전체에 공유합니다.
- 모든 답변은 한국어로 수행하며, 전문적이고 정중한 톤을 유지합니다.
